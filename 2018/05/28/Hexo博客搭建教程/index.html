Hexo搭建博客教程

发表于 2018-05-28 | 分类于 技术研究

优秀的人，都是孤独的

现在越来越多的人喜欢利用Github搭建静态网站，原因当然是简单省钱。本人也利用Github搭建了本博客，折腾博客的各种配置以及功能，占据里一些时间，俗话说：好记性不如烂笔头，在此详细记录下我是如何利用hexo+github搭建静态博客以及一些相关配置问题。

准备工作

  • 下载并安装node.js(官网去下载安装)，会默认安装npm
  • 下载安装git(官网去下载安装)
  • 下载安装hexo,方法：打开cmd运行npm install -g hexo（可能要翻墙）

本地搭建静态博客

  • 新建一个文件夹，如blog
  • 进入该文件夹内，右击运行git，输入：hexo init (生成hexo模板)
  • 生成完模板，运行npm install
  • 最后运行：hexo server（运行 程序，访问本地localhost:4000可以看懂啊博客已经搭建成功）

将博客与Github关联

  • 在Github上创建名字为XXX.github.io的项目，XXX为自己的 github用户名。

  • 打开本地的blog文件夹项目内的_config.yml配置文件，将其中的type设置为git(如下)

  • 运行：npm install hexo-deployer-git -save

  • 运行：hexo g（生成本地静态文件）
  • 运行：hexo d（将本地静态文件推送至Github）

此时，打开浏览器，访问https://marksmeng.github.io ，即可看到模板的样式。

更新博客内容

至此博客已经搭建完毕，接下来你就可以更新内容了

更新文章

  • 在blog目录下执行：hexo new “文章标题”,会在source->_ posts文件夹内生成一个.md文件
  • 编辑该文件（遵循Markdown规则）
  • 修改其实字段
    • title 文章的标题
    • data 创建日期
    • updated 修改日期
    • comments 是否开启评论（true or false）
    • tags 标签
    • catagories 分类
    • permalink url中的名字（文件名）
  • 编写正文内容
  • hexo clean 删除本地静态文件（Public 目录）,可以不执行
  • hexo g生成本地静态文件（Public目录）
  • hexo d将本地静态文件推送至github

添加菜单

进入theme目录，编辑_config.yml文件，找到menu字段，在该字段添加一个字段。

个性化设置

现在虽然能写博客了，可是界面依然有些丑，接下来让我们美化一下自己的博客

基本信息

在根目录（blog）下的_config.yml文件中可以修改标题，作者等信息。打开编辑该文件，注意：每个值的冒号后面都有一个半角的空格！

  • 未生效的写法：title:MarksMeng的博客
  • 能生效的写法：title:[空格]MarksMeng的博客
